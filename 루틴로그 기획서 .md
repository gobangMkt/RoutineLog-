# 루틴로그 기획서

<aside>
📌

이 문서는 **데일리 루틴 기록 앱(가칭: 루틴로그)** 기획서다.

- 목적: 사용자가 **날짜 단위로 투두를 기록**하고, 완료해도 **기록이 남는 형태로 관리**하며, 자주 쓰는 루틴을 **템플릿으로 저장/불러오기** 할 수 있게 한다.
- 기본 원칙: 완료 = 삭제가 아니라 **상태 변화(보임/스타일 변화)**가 기본값이며, 필요 시 **설정으로 ‘완료 시 숨김/삭제’**를 지원한다.
</aside>

## ✅ 0. 개요(필수)

---

<aside>
📌

### **개요(전체 내용 요약)***

- **한 줄 요약**: 날짜 기반으로 ‘큰 투두(프로젝트 단위) → 서브 투두’ 구조를 기록하고, 완료 이력은 남기되 사용자 설정에 따라 숨김/삭제 UX도 제공하는 루틴 앱
- **목적**: 데일리 루틴 실행/기록을 단순화하고, 반복 루틴은 템플릿으로 빠르게 재사용 가능하게 함
- **배경/가설**: 루틴 앱에서 가장 큰 이탈 원인은 “입력 피로도”와 “완료 후 기록 상실(히스토리 부재)”이며, ‘큰 투두-서브 투두’ 구조 + 템플릿 재사용이 이를 낮춘다
- **핵심 제안**: (1) 날짜(Day) 단위 화면, (2) 큰 투두(프로젝트)와 서브 투두 계층, (3) 완료해도 유지(기본), (4) 템플릿 저장/불러오기, (5) 완료 시 숨김/삭제 설정
- **성공 기준(Top 1~2)**: (1) D7 리텐션/재방문률 상승, (2) 1인당 일일 기록(생성) 투두 수 및 템플릿 사용률 상승
</aside>

## ✅ 1. 배경(선택)

---

### 1) 배경(현재 상황/문제/가설)

- **현재 상황(As-is)**
    - 투두 앱은 많지만, “루틴” 관점에서는 **매일 비슷한 묶음 입력**이 반복되어 입력 피로도가 큼
    - 일부 UX는 완료 시 항목이 사라져 **기록/회고 가치가 낮아짐**
- **문제 정의(왜 문제인가)**
    - 루틴은 “매일 실행하고 체크하는 행동”인데, 입력과 기록이 불편하면 지속이 어려움
    - 완료 히스토리가 없으면 성취감/회고/패턴 분석이 어렵고 장기 사용 동기가 떨어짐
- **원인 가설**
    1. 사용자는 할 일을 ‘단일 리스트’보다 **프로젝트/루틴 단위(큰 덩어리)**로 먼저 인지한다
    2. 완료 항목이 사라지면 “오늘 무엇을 해냈는지”가 남지 않아 회고 가치가 떨어진다
    3. 템플릿이 없으면 반복 입력 비용이 누적돼 기록을 포기한다

- **측정 지표(정의가 애매하면 산식까지)**

## ✅ 3. 목표 달성방법 (권장)

---

### 1) 핵심 개념/구조(정보 구조)

- **Day(날짜)**: 사용자의 기록 기준(예: 2026-04-23)
- **Parent Todo(큰 투두/프로젝트 단위)**: 날짜 안에서의 큰 덩어리(예: “마케팅 업무”, “운동 루틴”)
- **Sub Todo(투두 안의 투두)**: Parent 아래의 세부 할 일
- **Note(메모)**: 시간/체크 여부와 무관하게 남기는 텍스트 기록(선택)
    - 구현은 `type = todo | memo`로 통합하는 방식을 우선 고려

### 2) 핵심 실행 (Trigger → Action/CTA → Result)

#### [Trigger] 매일 기록해야 하는 순간

1. **가정**: 사용자는 매일 ‘해야 할 큰 덩어리’를 먼저 떠올리고, 세부 작업은 그 다음에 정리한다
2. **Trigger**: 날짜 화면 진입(오늘/내일) 또는 알림/루틴 실행 시간
3. **Action/CTA**: Day 화면에서 **Parent Todo를 먼저 추가** → 필요 시 내부에 Sub Todo 추가
4. **Result**: 생성까지 걸리는 시간 감소, 기록 지속률 증가

#### [Trigger] 완료 후에도 기록을 남기고 싶은 순간

1. **가정**: 사용자는 ‘완료=사라짐’보다 ‘완료=남아있되 표시만 바뀜’을 선호하는 경우가 많다
2. **Trigger**: 체크박스 탭(완료 처리)
3. **Action/CTA**: 기본값으로 **항목 유지 + 스타일 변경(취소선/회색) + doneAt 기록**
4. **Result**: 회고/성취감 강화, 루틴 지속성 증가

#### [Trigger] 완료하면 사라지는 UX를 원하는 사용자

1. **가정**: 일부 사용자는 화면이 깔끔해야 만족하며 완료 항목이 남아있으면 피로하다
2. **Trigger**: 설정 화면에서 선호도 선택
3. **Action/CTA**: 설정 토글 제공
    - 옵션 A(권장): **완료 시 자동 숨김(UI 필터)**
    - 옵션 B(선택): **완료 시 자동 삭제(복구 정책 필요)**
4. **Result**: 사용자 성향별 만족도 상승, 이탈 감소

#### [Trigger] 반복 루틴 입력 피로도가 누적되는 순간

1. **가정**: 반복되는 할 일 묶음은 템플릿으로 저장/적용할 수 있어야 한다
2. **Trigger**: 특정 날짜에 동일/유사 루틴을 다시 입력하려는 순간
3. **Action/CTA**: 템플릿 기능
    - **템플릿 저장**: 현재 Day의 Parent+Sub 묶음을 템플릿으로 저장(기본: 미완료만 포함 옵션)
    - **템플릿 불러오기**: 날짜를 선택해 템플릿을 **복제 생성**(합치기/덮어쓰기 옵션)
4. **Result**: 입력 시간 단축, 템플릿 사용률 증가

### 3) 시간(Time) 설계(요청 반영)

- **Parent Todo에서 시간 설정 가능(선택)**
    - 시간 있음: 시간대별 정렬/표시 가능(캘린더 느낌)
    - 시간 없음: 종일/언제든 영역(리스트)로 표시
- **Sub Todo는 기본적으로 시간 없음(선택적으로 확장)**
- **메모(Note)는 시간 입력 없이도 생성 가능**
    - UI에서 “시간 없음” 입력 흐름을 별도로 제공(예: ‘메모’ 탭)

## ✅ 4. 요구사항(R-*)(구현/정책까지 가면)

---

### 1) 요구사항(필요할 때만)

#### R-1. Day 단위 화면

- 목적: 날짜를 기준으로 루틴을 기록/조회
- 요구사항
    - 날짜 선택(오늘/내일/캘린더)
    - 해당 날짜의 Parent/Sub/Note 리스트 조회

#### R-2. Parent Todo(큰 투두) 생성/수정/삭제

- 목적: 큰 덩어리 단위로 하루를 구성
- 요구사항
    - Parent Todo 생성/수정
    - 시간 입력은 optional
    - 완료 체크 시 기본값은 **사라지지 않고 상태만 변경**

#### R-3. Sub Todo(투두 안의 투두) 생성/수정/삭제

- 목적: Parent Todo의 세부 실행 단위 기록
- 요구사항
    - Sub Todo 생성/수정
    - 완료 체크 시 기본값은 **사라지지 않고 상태만 변경**

#### R-4. 완료 처리 정책 + 설정

- 목적: 사용자 성향에 맞는 완료 UX 제공
- 요구사항
    - 설정: 완료 시 자동 숨김(기본 false)
    - 설정: 완료 시 자동 삭제(기본 false, 선택 기능)
- 예외/정책
    - 자동 삭제 제공 시, 최소한의 복구(Undo/휴지통) 정책 필요

#### R-5. 템플릿 저장/불러오기

- 목적: 반복 입력 비용 절감
- 요구사항
    - 템플릿 생성/수정/삭제
    - 템플릿 저장(현재 Day의 묶음을 저장)
    - 템플릿 불러오기(선택 날짜에 복제 생성)
    - 적용 옵션: 합치기/덮어쓰기, 시간 포함/제거

#### R-6. 메모(Note)

- 목적: 시간/체크와 무관하게 기록을 남기는 입력 흐름 제공
- 요구사항
    - Note 생성/수정/삭제
    - 체크박스 없음

## ✅ 5. 데이터/로그(측정 가능하게)

---

### 1) 데이터/로그(필요할 때만)

- 이벤트 로그(최소)
    - `day_viewed(date)`
    - `parent_created`, `parent_updated`, `parent_completed`
    - `sub_created`, `sub_updated`, `sub_completed`
    - `note_created`
    - `template_saved`, `template_applied`
    - `setting_changed(hideCompleted/deleteOnComplete)`
- 리포트
    - 날짜별 생성/완료 추이
    - 템플릿 사용률 및 템플릿별 사용 빈도
    - 설정 사용률(성향 분포)
