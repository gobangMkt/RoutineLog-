## ✅ 1. 공통사항

---

### 1) 접근 권한

- 로그인 사용자(개인 루틴/투두 데이터)

### 2) 적용 범위

- 모바일 앱(우선) + 웹(추후)
- 날짜(Day) 단위 기록/조회

### 3) 용어/정의(필요 시)

- **Day**: 날짜 컨테이너(예: 2026-04-23)
- **Parent Todo**: 큰 투두(프로젝트/루틴 단위)
- **Sub Todo**: Parent 내부 세부 투두
- **Note(메모)**: 체크 없이 텍스트 기록(시간 선택 가능)
- **Template**: Parent+Sub(또는 일부) 묶음을 저장해 특정 날짜에 복제 생성

## ✅ 2. 페이지/화면 구성

---

### 1) 페이지 헤더(Header)

| **구분** | **설명** |
| --- | --- |
| **전제 조건 및 기본값** |   • 기본 진입은 ‘오늘’ Day
  • 사용자의 완료 처리 설정값을 로드 |
| **구현 기능(노출/동작)** | **1. 노출사항**
  • 날짜(오늘/내일/캘린더 선택)
  • ‘템플릿 불러오기’ 버튼
  • (옵션) ‘설정’ 진입

**2. 노출 기준 및 동작 방식**
  • 날짜 변경 시 해당 Day의 Parent/Sub/Note를 조회해 리스트 갱신 |
| **에러 및 예외 처리** |   • 네트워크 오류 시: 재시도 안내 + 로컬 캐시(있다면) 표시 |
| **유효성 검사** | - |
| **데이터 출처 및 연동** |   • Day 기준 Task/Note/Template 조회 API |

### 2) 본문 섹션(Body)

**[섹션 1] Day 메인 리스트(Parent/Note)**

| **구분** | **설명** |
| --- | --- |
| **전제 조건 및 기본값** |   • 기본 정렬: (1) 시간이 있는 Parent/Note 먼저, (2) 시간 없는 항목은 하단 ‘종일/언제든’ 영역
  • 기본 필터: 완료 항목도 표시(설정에 따라 숨김 가능) |
| **구현 기능(노출/동작)** | **1. 노출사항**
  • Parent Todo 카드(제목, 시간 optional, 완료 상태)
  • Note 카드(체크박스 없음, 시간 optional)
  • ‘+ 추가’ CTA(빠른 입력)

**2. 동작 방식**
  • Parent 카드 탭 → 상세(서브 투두 리스트)로 진입
  • Note 탭 → 메모 편집 화면(또는 인라인 편집)
  • 완료 처리(체크) 시 동작은 설정값에 따름(아래 ‘완료 정책’ 참고) |
| **에러 및 예외 처리** |   • 저장 실패 시: 토스트 + 재시도 + 로컬 임시 상태 유지 |
| **유효성 검사** |   • Parent/Note 제목/내용은 빈 값 저장 불가(최소 1자) |
| **데이터 출처 및 연동** |   • Task/Note 목록 API(날짜 파라미터)
  • 정렬 키: timeStart, sortOrder |
</aside>

**[섹션 2] Parent Todo 상세(서브 투두)**

| **구분** | **설명** |
| --- | --- |
| **전제 조건 및 기본값** |   • Parent 1개당 Sub Todo 0~N개 |

  • Parent 제목/시간(옵션)
  • Sub Todo 리스트(체크박스 포함)
  • ‘서브 투두 추가’ 입력

  • Sub 완료 체크 시 기본값은 ‘사라지지 않고’ 스타일만 변경(설정에 따라 숨김/삭제 가능)
  • Parent 완료 체크 시: Parent만 완료 처리(서브 일괄 완료 여부는 정책 결정 필요) |
| **에러 및 예외 처리** |   • Parent 삭제 시: 서브 투두 처리 정책(기본: 함께 삭제 / 또는 고아 방지) |
| **유효성 검사** |   • Sub Todo 제목 빈 값 불가 |
| **데이터 출처 및 연동** |   • Sub Todo CRUD API(parentId) |
</aside>

**[섹션 3] 템플릿 저장/불러오기**

| --- | --- |
| **전제 조건 및 기본값** |   • 템플릿은 ‘투두 묶음(Parent+Sub)’ 중심

  • 템플릿 목록(이름, 포함 항목 수)
  • 템플릿 적용(불러오기) 버튼
  • 템플릿 저장 버튼(현재 Day에서)

**2. 동작 방식**
  • 불러오기: 선택 날짜에 템플릿을 ‘복제 생성’
  • 적용 옵션: (1) 합치기/덮어쓰기, (2) 시간 유지/제거

| **에러 및 예외 처리** |   • 덮어쓰기 선택 시: 확인 팝업(되돌리기 안내 포함) |
| **유효성 검사** |   • 템플릿 이름 1자 이상 필수
  • 템플릿 저장 시 최소 1개 항목 포함 필수 |
| **데이터 출처 및 연동** |   • Template CRUD API
  • Apply API(templateId, targetDate, mode) |

| **유효성 검사** | - |
| **데이터 출처 및 연동** |   • UserSettings API |

## ✅ 3. 상태/정책(필요 시)

---

### 1) 상태 정의

- Task 상태: `todo` / `done`
- (선택) 삭제 상태: `deleted`(휴지통) / `purged`(영구 삭제)

### 2) 권한/노출 정책

- 개인 데이터 기준(본인만 조회/수정)

### 3) ‘삭제/숨김/이력’ 정책

- 기본: 완료해도 유지 + doneAt 기록
- 숨김 모드: UI 필터로 숨김(데이터는 유지)
- 삭제 모드: 데이터 삭제(Undo 또는 휴지통 정책 권장)

## ✅ 4. 데이터/로그(측정 가능하게)

---

### 1) 이벤트 로그(최소)

- `day_viewed(date)`
- `parent_created(date, has_time)`
- `parent_completed(date, mode=keep|hide|delete)`
- `sub_created(parentId)`
- `sub_completed(parentId, mode=keep|hide|delete)`
- `note_created(date, has_time)`
- `template_saved(sourceDate, itemCount)`
- `template_applied(targetDate, templateId, mode=merge|overwrite)`
- `setting_changed(hideCompleted, deleteOnComplete)`

### 2) 데이터 연동/출처

- Task/Note/Template/Settings API

### 3) 리포트/지표

- D1/D7/D30 리텐션
- 사용자당 일일 생성/완료 수(Parent/Sub)
- 템플릿 사용률
- 완료 정책 선호 분포(keep/hide/delete)
